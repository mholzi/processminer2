# Step 2: Generate Suite Validation Report

## STEP GOAL:

Generate a comprehensive suite validation report with scoring, findings detail, coverage statistics, and remediation plan. Save to the process output folder.

## MANDATORY EXECUTION RULES (READ FIRST):

### Universal Rules:

- 🛑 NEVER skip any report section
- 📖 CRITICAL: Read the complete step file before taking any action
- 📋 Generate a complete, actionable report

### Step-Specific Rules:

- 🎯 Focus on clear, actionable output
- 📊 Include coverage statistics and health score
- 💡 Provide specific remediation steps for every finding

## MANDATORY SEQUENCE

### 1. Calculate Suite Health Score

Score the suite across six dimensions (each 0-100, weighted):

| Dimension | Weight | Scoring |
|-----------|--------|---------|
| Inter-Document References | 30% | (valid refs / total refs) * 100 |
| Sync Consistency | 20% | (synced pairs / total sync pairs) * 100 |
| Prerequisite Compliance | 15% | (met prerequisites / total prerequisites) * 100 |
| ID Uniqueness | 10% | 100 if no collisions, 0 if any |
| Confidence Propagation | 10% | (compliant sections / total propagation rules) * 100 |
| Coverage Completeness | 15% | (covered IDs / total Tier 1 IDs) * 100 |

**Suite Health Score** = weighted average of all dimensions

**Interpretation:**
- 90-100: Excellent — suite is internally consistent
- 70-89: Good — minor issues to address
- 50-69: Fair — significant gaps require attention
- Below 50: Poor — critical cross-document issues
