# Step 2: Generate Summary

## STEP GOAL:

Generate audience-appropriate executive summary and offer output options.

## MANDATORY EXECUTION RULES (READ FIRST):

### Universal Rules:

- 🛑 NEVER exceed configured length
- 📖 CRITICAL: Read the complete step file before taking any action
- 📋 YOU ARE A FACILITATOR creating communication

### Step-Specific Rules:

- 🎯 Focus on audience-appropriate content
- 🚫 FORBIDDEN to include technical jargon for executives
- 💬 Approach: Generate, preview, refine, save

## MANDATORY SEQUENCE

### 1. Extract Key Points

From loaded documents, extract:
- Process name, ID, scope
- Key metrics (step count, pain point count, etc.)
- Top findings by category
- Priority recommendations
- Quality/completion scores

### 2. Apply Audience Adaptation

**FOR Executive sponsors:**
- Focus on business impact
- Highlight ROI opportunities
- Keep technical details minimal
- Lead with value proposition

**FOR IT leadership:**
- Include system dependencies
- Highlight integration challenges
- Reference technical architecture
- Focus on implementation approach

**FOR Risk/Audit:**
- Emphasize control coverage
- Call out compliance gaps
- Include evidence requirements
- Focus on risk mitigation

"**Output options:**

### 6. Save and Confirm

**IF save selected:**

Write to: `{process_folder}/executive-summary.md`

"✓ **Executive Summary generated**

**File:** executive-summary.md
**Length:** {page_count} page(s) (~{word_count} words)
**Covers:** {summary_scope}

**What's next?**
- **[V]** View summary
- **[E]** Edit summary
- **[COMP]** Return to Companion"

#### Menu Handling Logic:

- IF V: Display full summary
- IF E: Allow edits, regenerate
- IF COMP: End workflow

#### EXECUTION RULES:

- ALWAYS halt and wait for user input
- Provide clear completion status
